## 一、整体架构

```
┌─ UI Layer ───────────────────────────────────────┐
│  Compose UI (悬浮窗 / 配置面板 / 日志面板)          │
│  BountyConfig → 用户勾选等级 + 目标次数             │
└───────────┬──────────────────────────────────────┘
            │ start / resume / stop
┌───────────▼──────────────────────────────────────┐
│  GameManager                                      │
│  ScriptState: IDLE / RUNNING / PAUSED             │
│  - 协程生命周期 (SupervisorJob + Dispatchers.Default) │
│  - MediaProjection 授权管理                        │
│  - 日志流 _logEvents → UI                         │
└───────────┬──────────────────────────────────────┘
            │ runLoop(configs)
┌───────────▼──────────────────────────────────────┐
│  WorkflowEngine (单循环流水线)                     │
│  GamePhase:                                       │
│  LOBBY → CHAT → RECRUIT_LIST(全流程) → TEAM_ROOM→│
│         FIGHT → SETTLEMENT → LOBBY                │
│  全局计数器: globalFailCount (3 次 → 停止 + dump)  │
└──┬──────────┬──────────┬──────────┬───────────────┘
   │capture   │detect    │click     │log
┌──▼──┐ ┌────▼────┐ ┌───▼────┐ ┌──▼──────────┐
│Screen│ │Scene    │ │Ninja   │ │LogUtil      │
│Capture││Detector │ │Accessi-│ │+ crash dump │
│(Media │ │(Template│ │bility  │ │(file)       │
│Proj.) │ │Matcher) │ │Service │ │             │
└──────┘ └─────────┘ └────────┘ └─────────────┘
```

### 组件职责

| 组件 | 职责 | 关键依赖 |
|------|------|---------|
| `GameManager` | 脚本生命周期、权限管理、日志分发 | `MediaProjection`, `CoroutineScope` |
| `WorkflowEngine` | 流水线状态机、异常兜底 | `ScreenCapture`, `SceneDetector`, `NinjaAccessibilityService` |
| `SceneDetector` | 模板缓存、阶段检测、等级图标匹配 | `OpenCV` (`TM_CCOEFF_NORMED`) |
| `TemplateMatcher` | OpenCV 模板匹配封装 | `OpenCV` (`Imgproc`, `Core`) |
| `ScreenCapture` | MediaProjection 截图 | `ImageReader`, `VirtualDisplay` |
| `NinjaAccessibilityService` | 模拟点击 | `GestureDescription` |

## 二、GamePhase 页面模型

### 页面定义

```kotlin
enum class GamePhase {
    IDLE,
    LOBBY,          // templates/lobby/
    CHAT,           // templates/chat/
    RECRUIT_LIST,   // templates/recruit_list/
    TEAM_ROOM,      // templates/team_room/
    FIGHT,          // templates/fight/
    SETTLEMENT,     // templates/settlement/
    RECOVERY,
    DONE
}
```

### 页面迁移

```
IDLE → LOBBY → CHAT → RECRUIT_LIST → TEAM_ROOM → FIGHT → SETTLEMENT → LOBBY...
  ↑                                                            │
  └────────────────────────────────────────────────────────────┘
  当 activeGrades.isEmpty() → DONE
```

### Phase → 方法映射

| Phase | 方法 | 返回目标 |
|-------|------|---------|
| `IDLE, LOBBY, CHAT, RECRUIT_LIST` | `phaseNavigateAndScan()` | `LOBBY / TEAM_ROOM / DONE` |
| `TEAM_ROOM` | `phaseValidate()` | `FIGHT / SETTLEMENT / LOBBY / DONE` |
| `FIGHT` | `phaseBattle()` | `SETTLEMENT / LOBBY / DONE` |
| `SETTLEMENT` | `phaseClaim()` | `IDLE / DONE` |
| `RECOVERY` | delay(1500ms) → | `IDLE` |

### GameContext

```kotlin
data class GameContext(
    currentPhase: GamePhase,
    activeGrades: List<BountyGrade>,          // 用户勾选的等级
    runCounts: MutableMap<BountyGrade, Int>,  // 各等级已跑次数
    targetRuns: Map<BountyGrade, Int>,        // 用户设定目标次数
    currentBounty: BountyGrade?,              // 当前处理中的等级（扫描循环核心标志位）
    totalCycles: Int                          // 总轮次
) {
    val allCompleted: Boolean
        // = activeGrades.all { runCounts[it] >= targetRuns[it] ?: it.defaultRuns }
}
```

## 三、核心参数

| 参数 | 值 | 作用于 | 说明 |
|------|-----|--------|------|
| `MAX_GLOBAL_FAIL` | 3 | 全局 | 累计异常上限，触发后停止并 dump |
| `NAVIGATE_RETRIES` | 15 | 导航 | 导航重试次数 (~15s 上限) |
| `POST_CLICK_DELAY` | 1000ms | 全局 | 点击后等待 UI 稳定 |
| `NORMAL_INTERVAL_MS` | 1000ms | 等待/结算 | 普通检测模式 |
| `FAST_INTERVAL_MS` | 100ms | 扫描/战斗 | 快速检测模式 |
| `LINEAR_MAX_MISS` | 3 | 结算 | 连续未检测触发兜底 |
| `LINEAR_MAX_LOOP` | 300 | 结算 | 单阶段最大循环数 |
| `SCAN_REFRESH_CYCLES` | 3 | 扫描 | 无匹配后触发 TAB 刷新 |
| `SCAN_MAX_REFRESH` | 5 | 扫描 | 最大 TAB 刷新次数 |
| `WAIT_BATTLE_TIMEOUT_MS` | 15000ms | 等待战斗 | 等待战斗超时 |

## 四、检测模式

### 检测速率

| 模式 | 周期 | 适用场景 |
|------|------|---------|
| 普通模式 | 1000ms | 导航、等待战斗、结算阶段 |
| 快速模式 | 100ms | 招募列表扫描（核心）、战斗检测 |

### 线性链检测 vs 全量扫描

| 特性 | 线性链检测 | 全量扫描 |
|------|-----------|---------|
| 触发时机 | 默认 | 兜底 / 异常恢复 |
| 检测范围 | 传入的 ScreenState 列表 | `detectionOrder` (所有状态) |
| 停止条件 | 第一个匹配 | 第一个匹配或全不匹配 |
| 方法 | `detectForPhase(screen, scope)` | `detectWithCoord(screen)` |

### 3-Miss 兜底机制

连续 N 次线性检测未匹配 → 触发一次范围判定:
- 扫描阶段: 3 周期无等级匹配 → TAB 刷新
- 结算阶段: 3 次 CONFIRM_BUTTON 未匹配 → SCOPE_CLAIM 判定
- 战斗阶段: 10 次无匹配 → SCOPE_BATTLE 判定

### 3-Strike 全局兜底

`globalFailCount` 累计异常 → 达到 3 次 → 脚本停止 + 崩溃日志写文件

## 五、异常处理链

```
WorkflowEngine.runLoop()
  └─ try { phaseXxx(ctx) } catch (非 CancellationException)
       └─ globalFailCount++
       └─ ctx.currentPhase = RECOVERY
       └─ RECOVERY → delay(1500ms) → IDLE
       └─ globalFailCount >= 3 → writeCrashLog() → 停止
```

## 六、服务层设计

### GameManager

```kotlin
object GameManager {
    state: StateFlow<ScriptState>  // IDLE / RUNNING / PAUSED
    logEvents: SharedFlow<String>  // 日志流 → UI

    fun toggleScript(context)  // start / pause / resume
    fun startScript(context)   // 等待 MediaProjection → 启动 WorkflowEngine
    fun pauseScript()          // cancel job → PAUSED
    fun resumeScript(context)  // 恢复 MediaProjection → 启动新 WorkflowEngine
    fun stopScript()           // cancel job → release MediaProjection → IDLE
    fun updateBountyConfigs(configs)
    fun getSelectedGrades(): List<BountyGrade>
}
```
