# UI 模块规格 - 悬浮窗/HUD/配置面板

## 悬浮窗系统

### 悬浮球

| 属性 | 值 | 说明 |
|------|-----|------|
| 尺寸 | 80x80dp | 圆形主球 |
| 初始位置 | 屏幕左侧边缘 | 球在左侧 |
| 拖拽 | 支持 | 可自由拖动 |
| 贴边吸附 | 支持 | 松手后自动吸附到最近的屏幕边缘 |
| 贴边隐藏 | 支持 | 无操作 5 秒后自动贴边，只露出 30px 小标签 |

#### 贴边隐藏后点击行为

悬浮窗贴边隐藏部分后，再次点击的交互分两步：

1. 第一次点击：恢复球到完整可见位置
2. 第二次点击：展开菜单

左侧贴边和右侧贴边均遵循此规则。

### 双菜单布局

悬浮球两侧各有一个菜单容器（`ll_menu_left` / `ll_menu_right`），通过 visibility 切换显示方向：

| 属性 | 值 | 说明 |
|------|-----|------|
| 布局方式 | 水平 LinearLayout，球在中间 | 菜单在球的左侧或右侧展开 |
| 菜单方向 | 由 `isBallOnRight()` 决定 | 球在右侧→菜单向左展开，反之向右 |
| 展开动画 | 菜单项依次滑入 | 每项延迟 60ms |
| 收起 | 点击菜单项或点击球 | 菜单收回 |

#### 收起交互要求

点击收回菜单时，根视图需要平滑过渡回球的吸附位置，不能出现跳跃闪烁。

### 菜单项

| 菜单项 | 功能 | 说明 |
|--------|------|------|
| 开始/停止 | 启动或停止脚本 | 调用 `GameManager.toggleScript()` |
| 日志 | 显示/隐藏日志面板 | 浮动日志窗口 |
| 设置 | 打开配置面板 | Compose 配置界面 |

### 日志面板

| 属性 | 值 | 说明 |
|------|-----|------|
| 位置 | 屏幕底部 | 可拖动 |
| 容量 | 最多30条 | 超出后旧日志被移除 |
| 样式 | 半透明背景，白色文字 | 等宽字体 |
| 自动滚动 | 是 | 新日志自动滚动到底部 |

## HUD 显示

### 进度显示

| 属性 | 值 | 说明 |
|------|-----|------|
| 显示条件 | `isOnGameDataPage = true` | 仅在大厅/招募列表节点显示 |
| 显示内容 | 当前悬赏等级 + 进度 | 如 "S级 2/5" |
| 位置 | 悬浮球旁边 | 随悬浮球移动 |
| 隐藏 | 脚本停止或非游戏页面 | 自动隐藏 |

### 等级显示顺序

| 业务线 | 显示顺序 | 说明 |
|--------|---------|------|
| 每日悬赏 | SS+ → SS → S → A → B → C → D | 最高等级在前 |
| 逆袭事件 | NSS+ → NS → NA | 事件等级在前 |

## 配置面板

### Compose UI 布局

| 组件 | 说明 |
|------|------|
| 悬赏等级勾选 | 多选框，勾选要自动完成的等级 |
| 最大次数设置 | 每个等级的最大完成次数 |
| 运行状态显示 | 当前脚本运行状态 |
| 启动/停止按钮 | 控制脚本启停 |

### 配置持久化

- 存储方式：SharedPreferences
- 配置读写：通过 `ScriptConfigRepository` 统一管理
- 线程安全：使用 StateFlow + snapshot() 模式

## 异常处理

| 异常 | 处理 |
|------|------|
| 悬浮窗权限被撤销 | 重新请求权限 |
| 配置读取失败 | 使用默认值 |
| 日志面板内存溢出 | 限制最多30条 |
